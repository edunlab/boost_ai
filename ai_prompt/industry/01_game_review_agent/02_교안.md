# 게임 유저 리뷰 분석 Agent 교안

## 도입

질문으로 시작한다.

- "게임 리뷰 1,000개가 하루에 올라온다면 운영팀은 무엇부터 봐야 할까요?"
- "칭찬 리뷰와 버그 제보, 과금 불만이 섞여 있을 때 어떤 기준으로 나눠야 할까요?"

핵심 메시지:

> 리뷰 분석 Agent의 목적은 리뷰를 예쁘게 요약하는 것이 아니라, 운영팀이 다음 행동을 결정할 수 있게 만드는 것이다.

## 핵심 설명

### 리뷰 분류 카테고리

- 버그/오류: 접속 불가, 튕김, 결제 오류, 보상 미지급
- 밸런스: 캐릭터 성능, 난이도, 매칭, 과도한 반복 플레이
- 과금: 가격, 확률형 아이템, 결제 만족도, 환불 요청
- 콘텐츠 요청: 신규 모드, 캐릭터, 이벤트, 스토리 추가 요청
- 칭찬: 그래픽, 스토리, 조작감, 운영 만족
- 기타/확인 필요: 의미가 불명확하거나 정보가 부족한 리뷰

### Agent 출력 구조

권장 출력 항목:

- 리뷰 요약
- 분류
- 감정 톤
- 긴급도
- 판단 이유
- 추천 후속 조치

## 실습 진행

1. 리뷰 샘플을 읽고 사람이 먼저 분류한다.
2. 분류 기준을 팀별로 작성한다.
3. Agent에게 분류 기준을 포함한 프롬프트를 작성한다.
4. 리뷰 5개를 입력해 결과를 확인한다.
5. 애매한 리뷰를 추가해 분류 기준을 보완한다.
6. 최종 결과를 운영 액션으로 변환한다.

## 강사용 피드백 포인트

- "부정" 하나로 뭉개지 말고 게임 운영 카테고리로 나누게 한다.
- 판단 이유를 반드시 출력하게 한다.
- 긴급도는 감정 세기가 아니라 운영 영향도를 기준으로 보게 한다.
- 리뷰가 복합적일 때 주분류와 부분류를 나누도록 유도한다.

## 마무리 질문

- 운영팀이 오늘 당장 확인해야 할 리뷰는 무엇인가?
- 자동 분류가 틀릴 가능성이 높은 리뷰는 어떤 유형인가?
- 이 Agent를 실제 운영 업무에 붙인다면 추가로 필요한 데이터는 무엇인가?
