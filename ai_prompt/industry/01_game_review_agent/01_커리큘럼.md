# 게임 유저 리뷰 분석 Agent 커리큘럼

## 과정 개요

- 주제: 게임 유저 리뷰/VOC 분석 Agent 설계
- 대상: 게임 기획, 운영, QA, 커뮤니티 관리에 관심 있는 학습자
- 목표: 유저 리뷰를 분류하고 우선순위를 판단하여 운영 액션으로 연결하는 Agent를 설계한다.
- 권장 시간: 3시간
- 주요 도구: Antigravity 또는 LLM 기반 Agent 설계 도구

## 학습 목표

- 게임 유저 리뷰를 버그, 밸런스, 과금, 콘텐츠 요청, 칭찬 등으로 분류할 수 있다.
- 분류 기준과 판단 이유를 프롬프트에 명확히 반영할 수 있다.
- 리뷰 분석 결과를 운영 우선순위와 대응 액션으로 변환할 수 있다.
- 리뷰 분석 Agent의 입력, 처리, 출력 구조를 설계할 수 있다.

## 차시 구성

| 단계 | 주제 | 핵심 활동 | 산출물 |
|---:|---|---|---|
| 1 | 게임 VOC 이해 | 리뷰 유형과 운영 의사결정 연결 | 리뷰 분류 기준표 |
| 2 | 분류 프롬프트 설계 | 카테고리, 판단 이유, 긴급도 정의 | 분류 프롬프트 |
| 3 | Workflow 설계 | 리뷰 입력 → 분류 → 우선순위 → 액션 제안 | Agent 흐름도 |
| 4 | 테스트와 개선 | 애매한 리뷰, 복합 리뷰, 과격 표현 테스트 | 개선 프롬프트 |
| 5 | 발표 | 운영팀 관점의 활용 방안 공유 | 1페이지 설계 요약 |

## 권장 실습 시나리오

모바일 RPG 게임 운영팀이 앱스토어와 커뮤니티 리뷰를 매일 확인해야 한다. 리뷰가 많아 사람이 모두 읽기 어렵기 때문에, Agent가 리뷰를 자동 분류하고 긴급도를 판단한 뒤 운영팀이 확인해야 할 이슈를 요약한다.

## 최종 산출물

- 리뷰 분류 기준표
- 리뷰 분석 Agent 프롬프트
- 리뷰별 분류 결과표
- 운영 액션 제안표
- Agent 설계 요약서

## 평가 기준

| 항목 | 기준 |
|---|---|
| 분류 기준 | 게임 운영 맥락에 맞는 카테고리를 정의했는가 |
| 판단 이유 | 분류 결과에 대한 설명이 납득 가능한가 |
| 우선순위 | 긴급도와 영향도를 고려했는가 |
| 실행 가능성 | 운영팀이 바로 사용할 수 있는 액션을 제안했는가 |
