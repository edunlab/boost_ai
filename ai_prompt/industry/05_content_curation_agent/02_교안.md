# 콘텐츠 추천 큐레이션 Agent 교안

## 도입

질문으로 시작한다.

- "좋은 추천은 인기순 목록과 무엇이 다를까요?"
- "사용자가 왜 이 추천을 받았는지 이해하면 만족도가 높아질까요?"

핵심 메시지:

> 추천 Agent는 정답을 맞히는 도구가 아니라, 사용자의 취향과 상황에 맞는 선택지를 설명 가능하게 제안하는 도구다.

## 핵심 설명

### 취향 입력 요소

- 선호 장르
- 싫어하는 요소
- 이용 시간
- 예산
- 혼자/함께 여부
- 원하는 감정
- 최근 좋아한 콘텐츠

### 추천 출력 요소

- 추천 항목
- 추천 이유
- 적합한 상황
- 주의할 점
- 대체 추천
- 연결 상품 또는 후속 콘텐츠

### 조건 분기 예시

- 시간이 짧으면 가벼운 콘텐츠 추천
- 예산이 낮으면 무료 또는 저가 상품 추천
- 공포를 싫어하면 스릴러 강도를 낮춘 대체안 추천
- 친구와 함께라면 협동 게임이나 대중적인 영화 추천

## 실습 진행

1. 사용자 프로필 샘플을 읽는다.
2. 추천 조건을 구조화한다.
3. 게임, 영화, 상품 추천 기준을 작성한다.
4. 큐레이션 세트를 생성한다.
5. 추천 이유와 제외 이유를 추가한다.
6. 다른 사용자 프로필로 테스트하고 결과 차이를 비교한다.

## 강사용 피드백 포인트

- 단순히 유명한 콘텐츠를 추천하지 않도록 한다.
- 추천 이유가 사용자 입력과 직접 연결되게 한다.
- 싫어하는 요소와 제약 조건을 반드시 반영하게 한다.
- 커머스 상품은 콘텐츠 경험을 보완하는 역할로 연결하게 한다.

## 마무리 질문

- 추천 결과가 사용자 상황을 충분히 반영했는가?
- 추천 이유 없이 결과만 제시하면 어떤 문제가 생길까?
- 실제 서비스에서 추천 Agent가 실수하면 어떤 위험이 있을까?
