# 에듀니티랩 1,2일차 실습교육 커리큘럼

## 1. 과정 개요

- 과정명: [중급 공통] Antigravity 기반 AI 워크플로우 설계와 에이전트 구축
- 교육 형태: ABL(실습 중심 학습)
- 교육 대상: 비전공/전공 혼합
- 난이도: 중급
- 교육 방식: 오프라인, 이론 및 실습 병행
- 필요 도구: 1인 1PC, 인터넷 연결, Antigravity 활용 환경
- 본 문서 범위: 1,2일차 6차시 실습교육 설계

본 과정은 AI, LLM, Agent의 차이를 이해하고, 단일 프롬프트를 넘어 워크플로우 기반의 문제 해결 방식으로 확장하는 것을 목표로 한다. 학습자는 역할 기반 프롬프트, 출력 구조화, 단계형 처리, 조건 분기, 업무 자동화 설계를 실습하며 Mini Project를 통해 실제 업무 문제를 Agent 형태로 설계한다.

## 2. 교육 목표

교육 후 학습자는 다음을 수행할 수 있다.

- AI, LLM, Agent의 개념과 차이를 설명할 수 있다.
- Agent를 단순 질의응답 도구가 아닌 실행 중심 시스템으로 인식할 수 있다.
- Role, Instruction, Context, Output을 활용해 구조화된 프롬프트를 설계할 수 있다.
- 표, Bullet, JSON 등 목적에 맞는 출력 형식을 지정할 수 있다.
- Prompt Chaining을 활용해 문제를 단계형 Workflow로 분해할 수 있다.
- 조건 분기와 판단 로직을 적용해 상황별 결과를 생성하는 Agent를 설계할 수 있다.
- 반복 업무를 입력, 처리, 출력 구조로 정리하고 자동화 가능성을 판단할 수 있다.
- Mini Project 설계서를 작성하고 팀별 발표와 피드백을 수행할 수 있다.

## 4. 차시별 상세 커리큘럼

### 1차시. AI · LLM · Agent 개념 이해

- 학습 목표:
  - AI, LLM, Agent의 차이를 이해한다.
  - Agent를 실행 시스템으로 인식한다.
- 주요 내용:
  - AI, LLM, Agent의 관계
  - 생성형 AI와 Agent의 차이
  - Agent 구성요소: Input, Reasoning, Action, Output
- 실습:
  - 동일 질문을 일반 질문, 역할 기반 질문, 실행 요청형 질문으로 입력한다.
  - 결과 차이를 비교하고 왜 달라졌는지 토론한다.
- 산출물:
  - 질문 방식별 응답 비교표
  - Agent 관점에서 개선한 질문 1개

### 2차시. 프롬프트 구조화 & Persona 설계

- 학습 목표:
  - 프롬프트를 구조적으로 설계할 수 있다.
  - 역할(Persona)에 따른 결과 차이를 이해한다.
- 주요 내용:
  - Prompt 구성요소: Role, Instruction, Context, Output
  - Persona 설계 방법
  - 역할에 따른 응답 변화 원리
- 실습:
  - 동일 질문을 상담원, 기획자, 전문가 Persona로 실행한다.
  - 가장 효과적인 Persona 설계 사례를 발표한다.
- 산출물:
  - Persona 3종 프롬프트
  - Persona별 응답 비교표
  - 최종 개선 프롬프트

### 3차시. 출력 구조 제어 & 데이터화

- 학습 목표:
  - 출력 형식을 제어할 수 있다.
  - 텍스트를 구조화된 데이터로 변환할 수 있다.
- 주요 내용:
  - Output Format 제어: 표, Bullet, JSON
  - 정보 추출 및 구조화 방법
  - AI 결과를 업무 데이터로 활용하는 관점
- 실습:
  - 회의록을 표, Bullet, JSON 형식으로 변환한다.
  - 가독성과 활용성 기준으로 결과를 비교한다.
- 산출물:
  - 회의록 요약 표
  - JSON 변환 결과
  - 형식별 장단점 메모

### 4차시. Agent 흐름 설계: Prompt에서 Workflow로

- 학습 목표:
  - 단일 프롬프트를 단계형 구조로 확장할 수 있다.
  - Workflow 개념을 이해한다.
- 주요 내용:
  - Prompt Chaining 개념
  - Task, Step, Flow 구조
  - 단계별 처리 방식
- 실습:
  - 요약, 핵심 추출, 액션 도출의 3단계 Workflow를 구성한다.
  - 단계 설계 방식과 결과 차이를 비교한다.
- 산출물:
  - 3단계 Workflow 설계표
  - 단계별 프롬프트
  - 최종 액션 아이템 결과

### 5차시. 조건 분기 & 판단형 Agent 설계

- 학습 목표:
  - 조건 분기(if/else)를 설계할 수 있다.
  - 판단 기반 Agent를 구현할 수 있다.
- 주요 내용:
  - 조건 분기 구조 설계
  - 분류 및 판단 로직 설계
  - 예외 처리 개념
- 실습:
  - 리뷰 분류 Agent를 설계한다.
  - 리뷰를 긍정, 부정, 문의로 분류하고 이유를 함께 출력한다.
  - 분류 기준과 정확도를 비교한다.
- 산출물:
  - 분류 기준표
  - 리뷰 분류 결과표
  - 예외 입력 대응 규칙

### 6차시. 업무 자동화 Agent 설계: Mini Project

- 학습 목표:
  - 실제 업무를 Agent로 설계할 수 있다.
  - 자동화 대상 선정 기준을 이해한다.
- 주요 내용:
  - 자동화 대상 선정 기준: 반복성, 규칙성, 데이터성
  - 입력, 처리, 출력 구조 설계
  - Agent 설계서 작성 방법
- 실습:
  - 개인 또는 팀 단위로 Agent 설계서를 작성한다.
  - 문제 정의, 흐름, 기대 효과를 발표하고 피드백을 받는다.
- 산출물:
  - Mini Project Agent 설계서
  - 발표용 1페이지 요약
  - 피드백 반영 개선안

## 6. 운영 팁

- 비전공자가 포함된 과정이므로 용어 설명은 짧게 하고 즉시 비교 실습으로 연결한다.
- 매 차시마다 "좋은 결과"보다 "왜 결과가 달라졌는지"를 말하게 한다.
- 실습 산출물은 모두 Mini Project 설계서의 재료가 되도록 연결한다.
- 2일차 마지막에는 완성도보다 문제 정의, 입력/처리/출력 구조, 자동화 적합성에 초점을 둔다.
