도입 질문: 질문과 설계의 차이
1일차 에듀니티랩
  • 도입 질문: "ChatGPT에게 매번 좋은 답변을 얻기 위해 프롬프트를 고쳐 쓰는 것과, 하나의 에이전트를 설계하는 것은 무엇이 다를까요?"
  • 비교 분석:
    • ChatGPT 질문: 일회성 질의, 답변을 얻기 위한 즉흥적 시도, 개인의 프롬프트 작성 능력에 의존.
    • Agent 설계: 반복 가능한 시스템 구축, 입력을 해석해 결과를 내는 자동화 파이프라인 형성, 조직의 표준 워크플로우 적용.
우리는 프롬프트 엔지니어링이라고 하면 매번 다르게 입력해서 좋은 대답을 구하는 '개인 비서와의 대화'를 생각합니다. 하지만 에이전트 설계는 다릅니다. 사용자가 어떤 질문을 던지더라도 정해진 프로세스에 맞춰 판단하고, 도구를 실행해 표준화된 업무 결과물을 뽑아내는 '시스템'을 짓는 일입니다.
화면을 2분할하여 왼쪽은 '대화형 UI(Chat)'를, 오른쪽은 '기어와 파이프라인(System)' 아이콘을 배치하여 대비 효과 극대화.

도입 질문: AI가 일을 처리하려면?
1일차 에듀니티랩
1
판단 (Reasoning)
현재 입력된 데이터가 무엇인지 파악하고 분류하는 문제 정의 및 의사결정 단계
2
도구 사용 (Action)
메일 발송, DB 저장, API 통신 등 외부 시스템과의 연결 및 동작 수행 단계
3
구조화 출력 (Output)
지정된 템플릿이나 JSON 포맷으로 최종 결과를 전달하여 후속 자동화 연계 보장
'판단 -> 도구 사용 -> 출력'의 유기적인 순서도를 가로형 인포그래픽으로 구현.
1차시. AI · LLM · Agent 개념 이해
핵심 설명: AI · LLM · Agent의 관계
1일차 에듀니티랩
AI (인공지능)
인간의 지능적 행동을 컴퓨터 시스템으로 구현하려는 광범위한 과학 기술 숲
LLM (대형 언어 모델)
방대한 텍스트 데이터를 통해 자연어 맥락과 연산 능력을 극대화한 핵심 언어 두뇌 엔진
Agent (에이전트)
LLM을 중추신경으로 삼아 스스로 판단(Reasoning)하고 도구를 활용해 행동(Action)하는 자동화 시스템
세 개의 원이 서로를 포함하고 있는 벤다이어그램(AI > LLM > Agent)을 입체적인 레이어로 표현.
1차시. AI · LLM · Agent 개념 이해
핵심 설명: 생성형 AI vs 에이전트의 차이점
1일차 에듀니티랩
  • 차이 비교표:
구분 생성형 AI (Generative AI) AI 에이전트 (AI Agent)
동작 방식 사용자 질문에 대해 즉시 답변 생성 목표가 주어지면 내부 계획을 수립하여 다단계로 실행
사용 주체 인간이 직접 개입하여 질문을 보정함 스스로 판단하고 도구를 호출하며 인간은 감독함
산출물 자연어 줄글, 일반적 이미지 등 데이터베이스 레코드, 이메일 전송, 완료 보고서 등
단순 생성형 AI는 질문을 던지면 바로 답을 주는 단발성 관계입니다. 반면 에이전트는 '이 고객의 환불 요청 건을 최종 처리해줘'라는 목표를 주면, 규칙에 따라 환불 사유를 분류하고, 정책을 확인한 후, 환불 메일을 발송하는 단계를 스스로 기획하여 완료합니다.
두 방식의 데이터 흐름 차이를 단순 선형 흐름(생성형 AI)과 순환형 루프(에이전트) 다이어그램으로 대조하여 시각화.
1차시. AI · LLM · Agent 개념 이해
핵심 설명: Agent 4대 구성요소 (IRAO)
1일차 에듀니티랩
I
Input (입력)
사용자 발화, 이메일 본문, API 호출 등으로 전달받은 원시 데이터
R
Reasoning (판단)
페르소나 규칙에 따라 입력 데이터 분석 및 실행 계획 수립
A
Action (도구)
검색, API 요청, 코드 실행 등 물리적 태스크 수행
O
Output (출력)
최종 JSON, 사용자 알림 등 포맷팅된 결과 전달
IRAO 4개 영역을 각각의 카드 레이아웃으로 구분하고 흐름을 화살표로 연결하여 표현.

  • 준비 도구: Antigravity 프롬프트 테스트 환경

이어서 첫 번째 실습을 진행하겠습니다. 동일한 고객 컴플레인 상황에서 질문을 단순하게 했을 때, 역할을 부여했을 때, 그리고 일의 순서와 형식을 명확히 지정해 주었을 때 LLM이 내놓는 답의 현실성과 비즈니스 가치가 얼마나 달라지는지 확인해 보겠습니다.
실습 과정을 1, 2, 3 단계 아이콘과 함께 직관적으로 정리한 타임라인 그래픽 제공.
1차시. AI · LLM · Agent 개념 이해
실습 진행: 3개 유형 프롬프트 대조
1일차 에듀니티랩
1단계: 일반 질문 
고객 불만 메일에 답장해줘.
2단계: 역할 지정 
너는 친절한 고객지원 담당자야. 고객 불만 메일에 답장해줘.
3단계: 구조화 질문 
고객 불만 메일을 분석하고, 불만 원인, 사과 문장, 해결 방안, 후속 조치로 나누어 답장 초안을 작성해줘.
3개 프롬프트 상자를 강조 효과와 함께 나열하고, 각 단계마다 늘어나는 구체적 제약 조건을 표시.

공유 및 토론: 무엇이 결과의 차이를 만드는가?
1일차 에듀니티랩
  • 공유 질문:
    • 1. 어떤 방식의 질문 결과가 실제 현업 메일로 즉시 사용하기에 가장 수월했습니까?
    • 2. 1단계와 3단계 결과물의 퀄리티 차이를 만든 핵심적인 요인은 무엇입니까?
    • 3. 에이전트 설계라는 관점에서 볼 때, 'Action(실행)' 단계에 활용될 수 있는 부분은 무엇입니까?
실습 결과를 서로 나누어 보겠습니다. 단순히 친절하게 답하라는 지시보다, 메일의 본질적인 구조(사과, 원인 분석, 대안 제시)를 분할하여 지시했을 때 훨씬 정돈된 대답이 나옵니다. 에이전트 설계의 핵심은 바로 이 '단계적 지시'와 '구조 설계'에 있습니다.
토론 분위기를 돋우는 말풍선 그래픽과 대표 질문 3가지를 가독성 높게 배치.
