핵심 설명: AI · LLM · Agent의 관계
1일차 에듀니티랩
AI (인공지능)
인간의 지능적 행동을 컴퓨터 시스템으로 구현하려는 광범위한 과학 기술 숲
LLM (대형 언어 모델)
방대한 텍스트 데이터를 통해 자연어 맥락과 연산 능력을 극대화한 핵심 언어 두뇌 엔진
Agent (에이전트)
LLM을 중추신경으로 삼아 스스로 판단(Reasoning)하고 도구를 활용해 행동(Action)하는 자동화 시스템
세 개의 원이 서로를 포함하고 있는 벤다이어그램(AI > LLM > Agent)을 입체적인 레이어로 표현.

핵심 설명: 생성형 AI vs 에이전트의 차이점
1일차 에듀니티랩
  • 차이 비교표:
구분 생성형 AI (Generative AI) AI 에이전트 (AI Agent)
동작 방식 사용자 질문에 대해 즉시 답변 생성 목표가 주어지면 내부 계획을 수립하여 다단계로 실행
사용 주체 인간이 직접 개입하여 질문을 보정함 스스로 판단하고 도구를 호출하며 인간은 감독함
산출물 자연어 줄글, 일반적 이미지 등 데이터베이스 레코드, 이메일 전송, 완료 보고서 등
단순 생성형 AI는 질문을 던지면 바로 답을 주는 단발성 관계입니다. 반면 에이전트는 '이 고객의 환불 요청 건을 최종 처리해줘'라는 목표를 주면, 규칙에 따라 환불 사유를 분류하고, 정책을 확인한 후, 환불 메일을 발송하는 단계를 스스로 기획하여 완료합니다.
두 방식의 데이터 흐름 차이를 단순 선형 흐름(생성형 AI)과 순환형 루프(에이전트) 다이어그램으로 대조하여 시각화.

import openai

def handle_email_pipeline(user_email):
    # Task 1: 메일 분석
    analysis_prompt = f"이메일을 카테고리화 하세요: {user_email}"
    res1 = openai.chat.completions.create(model="gpt-4o", messages=[{"role": "user", "content": analysis_prompt}])
    category = res1.choices[0].message.content
    
    # Task 2: 초안 작성
    reply_prompt = f"이메일 본문 '{user_email}'에 대해 카테고리 '{category}'에 맞춰 답변 초안을 작성하시오."
    res2 = openai.chat.completions.create(model="gpt-4o", messages=[{"role": "user", "content": reply_prompt}])
    return res2.choices[0].message.content

        raw_text = response.choices[0].message.content
        try:
            # JSON 검증 시도
            data = json.loads(raw_text)
            return data
        except json.JSONDecodeError as e:
            print(f"Error parsing JSON: {e}. Retrying...")
            # 에러 피드백을 전달하여 다시 생성을 요청함
            current_prompt = f"{prompt}\n\n[이전 시도 에러 발생]: {e}\n위의 에러를 반영하여 완벽한 JSON을 다시 작성하시오."
    raise Exception("Max retries reached without valid JSON output.")
5개의 리뷰 텍스트 상자 옆에 각각 카테고리 꼬리표(Tag)가 정답 가이드라인과 매칭되는 시각적 구성.
