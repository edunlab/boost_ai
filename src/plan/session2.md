# [교안] 2차시. 프롬프트 구조화 & Persona 설계

## 1. 개요 및 목표

- **대상**: AI 에이전트의 판단력을 고도화하고자 하는 학습자
- **시간**: 총 60분 (도입 5분, 설명 20분, 실습 25분, 공유 및 정리 10분)
- **목표**:
  - 페르소나가 답변의 톤앤매너와 전문성에 미치는 영향의 원리를 이해한다.
  - 글로벌 표준 프롬프트 프레임워크인 RICO 구조를 설계할 수 있다.
  - 단순 역할을 넘어 구체적인 의사결정 기준이 포함된 최적의 페르소나를 조립한다.

## 3. 세부 강의 가이드

### [00:00 - 00:05] 도입: 생각 열기 (5분)

#### 강사 설명 가이드
> "지나가는 사람에게 갑자기 '블로그 글 하나 써줘'라고 하면 어떤 글이 나올까요? 평범한 글이겠죠. 하지만 '10년 차 IT 전문기자님, 비전공자 직장인을 타겟으로 인공지능 트렌드 칼럼을 써주세요'라고 부탁하면 결과물은 완전히 달라집니다. 인공지능에게도 마찬가지로 구체적인 '신분'과 '규칙'을 주는 페르소나 설계가 필수적입니다."

#### 도입 질문
- *"같은 주제로 물어봐도 상담원과 개발자가 답변하는 내용은 어떻게 달라질까요?"*
- *"AI가 단순한 답변 말투만 흉내 내는 것을 넘어 전문 지식을 활용하도록 하려면 무엇을 명시해야 할까요?"*

### [00:05 - 00:25] 핵심 설명: RICO 구조와 Persona 설계 법칙 (20분)

#### 1) 글로벌 표준 프롬프트 프레임워크: RICO (R-I-C-O)
프롬프트를 작성할 때 줄글로 길게 쓰는 것보다, 4가지 핵심 요소를 구분하여 적는 것이 효율적입니다.

- **Role (역할)**: AI의 신분 설정 (예: 10년 차 금융 규제 준법감시관)
- **Instruction (지시사항)**: 해야 할 명확한 행동 (예: 이번 신규 약관의 위법성 검토)
- **Context (맥락/배경)**: 업무 배경 및 제약 사항 (예: 최신 금융위원회 가이드라인 준수)
- **Output (출력 형식)**: 최종 양식 및 말투 지정 (예: Markdown 표 형식, 전문적이고 정중한 어조)

#### 시각 자료 1. RICO 프레임워크 구조도
```
+-------------------------------------------------------------+
|                     RICO PROMPT FRAMEWORK                   |
+-------------------------------------------------------------+
|  [ R ] Role       : AI의 가상 인격과 직업 정의              |
|  [ I ] Instruction: 수행해야 할 주요 업무 및 지침           |
|  [ C ] Context    : 제약 조건, 관련 데이터 및 배경 정보      |
|  [ O ] Output     : 결과물의 톤앤매너와 데이터 포맷(형식)    |
+-------------------------------------------------------------+
```
*그림 설명: 4개의 퍼즐 조각이 맞물려 하나의 완성된 프롬프트 카드를 형성하는 입체 인포그래픽을 묘사합니다.*

#### 2) 좋은 페르소나 설계를 위한 3가지 황금 규칙
1. **역할을 아주 구체적으로 설정할 것**: 단순 '마케터'보다 '퍼포먼스 마케팅 대행사 대표'처럼 구체적으로 지정합니다.
2. **판단 및 의사결정 기준을 심어줄 것**: "마음대로 답해줘"가 아닌 "회사 내부 규정집 A항에 근거해 판단해줘"와 같이 내부 규칙을 줍니다.
3. **타겟 독자(Audience)를 지정할 것**: 최종 글을 읽을 사람의 수준을 설정해야(예: IT 지식이 없는 50대 은퇴 예정자) 알맞은 용어가 사용됩니다.

#### 시각 자료 2. 가중치 활성화 다이어그램
```
         [ LLM 거대 지식 신경망 ]
           /        |        \
[마케팅 영역]   [코딩 영역]   [법률 영역]
     (★)            ( )          ( )
      |
      +──> '10년 차 마케팅 팀장' 페르소나 설정 시 해당 영역 가중치 집중 활성화!
```
*그림 설명: LLM 신경망 내에서 지정된 페르소나(예: 마케팅)에 불이 들어와 활성화되는 모습을 시각화하여, 왜 페르소나 지정이 답변 전문성을 높이는지 과학적으로 이해하도록 돕습니다.*

### [00:25 - 00:50] 실습 진행: 페르소나별 출시 안내문 비교 및 융합 (25분)

#### 실습 업무 시나리오
```text
신규 AI 서비스 'Antigravity'가 출시됩니다.
- 타겟: 중소기업 마케팅 담당자
- 주요 기능: AI 기반 마케팅 콘텐츠 자동 생성
- 목적: 서비스 출시 안내문 작성
```

#### 실습 수행 단계
1. **Persona A (상담원)**: 친절하고 혜택 중심으로 작성하도록 프롬프트를 실행합니다.
2. **Persona B (기획자)**: 기능의 상세 정보와 기술적 특징 위주로 작성하도록 실행합니다.
3. **Persona C (트렌드 분석가)**: 시장의 흐름과 투자 대비 효과(ROI) 위주로 작성하도록 실행합니다.
4. **최종 융합 프롬프트 작성**: 3개 페르소나의 장점을 조합한 나만의 하이브리드 프롬프트(예: "기획자의 전문 지식과 상담원의 친절함을 결합한 마케팅 실장")를 조립하여 테스트합니다.

#### 강사용 팁 (Instructor Tip)
> "학습자들이 단지 결과물만 복사해 붙여넣는 것이 아니라, 각 페르소나가 문장에서 첫머리를 어떻게 여는지, 어떤 단어에 집중하는지 차이점을 발견할 수 있도록 안내하세요."

### [00:50 - 01:00] 정리 및 토론 (10분)

#### 공유 질문 (토론 거리)
- "단순히 '역할'만 줬을 때와 '판단 기준(Context)'을 함께 줬을 때 답변 품질에 어떤 차이가 있었나요?"
- "실무에서 본인의 업무에 바로 대입해 볼 수 있는 가장 매력적인 페르소나 조합은 무엇인가요?"

#### 심화 팁: 페르소나 일관성 유지법 (Jailbreak 방지)
- 프롬프트 마지막에 **"당신은 지정된 역할을 절대 잊어서는 안 되며, 본인 분야가 아닌 질문에는 솔직히 모른다고 답해야 합니다"** 같은 예외 처리 가이드라인을 삽입하는 팁을 안내합니다.

#### 강사 마무리 요약 스크립트
> "페르소나는 단순한 말투 흉내가 아니라 답변의 **'관점과 전문 지식 수준'**을 고정하는 힘입니다. 다음 3차시에는 이렇게 도출된 고품질의 답변을 엑셀이나 시스템에서 바로 활용할 수 있도록 딱 맞는 규격으로 인출하는 '출력 구조 제어'를 실습해 보겠습니다."
