# [교안] 3차시. 출력 구조 제어 & 데이터화

## 1. 개요 및 목표

- **대상**: AI 에이전트의 응답을 외부 프로그램이나 실무 양식에 자동 연동하고자 하는 학습자
- **시간**: 총 60분 (도입 5분, 설명 20분, 실습 25분, 공유 및 정리 10분)
- **목표**:
  - 줄글 응답의 업무적 한계를 인지하고, 규격화된 출력 포맷의 장점을 이해한다.
  - 3대 비즈니스 출력 포맷(표, Bullet Point, JSON)의 활용처를 설명할 수 있다.
  - Few-shot 예시 기법을 사용하여 AI의 불필요한 서두/설명을 제거하고 구조를 강제할 수 있다.

## 3. 세부 강의 가이드

### [00:00 - 00:05] 도입: 생각 열기 (5분)

#### 강사 설명 가이드
> "회의를 녹음해서 텍스트로 받아 적었는데, 이 파일이 30페이지 분량입니다. 부장님이 '핵심만 요약해서 엑셀 표로 정리해 줘'라고 하시면, 요약은 잘해주는 AI 결과물을 보고 다시 한 땀 한 땀 엑셀에 타이핑해야 하는 불편함이 생기죠. 왜 그럴까요? 바로 출력 구조가 제어되지 않았기 때문입니다. 오늘은 이 출력 포맷을 완벽하게 다스리는 기법을 배웁니다."

#### 도입 질문
- *"AI가 준 훌륭한 문장을 복사해서 내 보고서나 시스템에 바로 붙여넣기 힘든 진짜 이유가 무엇일까요?"*
- *"답변 내용뿐만 아니라 답변의 '모양(규격)'까지 AI에게 지시해 본 경험이 있으신가요?"*

### [00:05 - 00:25] 핵심 설명: 출력 구조 제어 기법 (20분)

#### 1) 3대 출력 포맷의 쓰임새
- **표 (Table)**: 항목 간의 비교, 데이터 정합성 확인, 일정 및 담당자 관리에 적합 (인간의 보고서용)
- **불릿 (Bullet Point)**: 신속한 전파, 핵심 이슈 요약, 짧은 브리핑에 적합 (인간의 메신저용)
- **JSON**: 데이터베이스(DB) 저장, 다른 에이전트 시스템에 파라미터 전달용 (컴퓨터/시스템 연동용)

#### 시각 자료 1. 비정형 텍스트 필터 파이프라인
```
[비정형 긴 줄글 회의록]
         │
         ▼ (LLM 출력 필터링)
+───────────────────────────────────────────────────+
| [표 형식]          [불릿 형식]          [JSON 형식]  |
|  구분 | 담당 | 기한 │ - 결론 1           │  "tasks": [ │
|  안내 | 이대리| 5/20 │ - 실행사항 1      │    {"owner":..│
+───────────────────────────────────────────────────+
```
*그림 설명: 뒤죽박죽 섞여 있는 회의록 줄글이 필터를 통과하면서 표, 불릿, JSON의 세 갈래 깔끔한 데이터 카드로 매핑되어 변환되는 그림입니다.*

#### 2) 출력 제어를 위한 4대 프롬프트 법칙
1. **스키마(Schema) 정의**: 채워야 할 열 이름이나 JSON 키를 영어로 명확히 정해줍니다.
2. **빈 값 처리 규칙 지정**: "정보가 없으면 공백으로 두지 말고 'N/A'라고 적을 것"처럼 빈틈을 막습니다.
3. **불필요한 인사말 차단**: `"네, 요청하신 내용을 정리해 드리겠습니다"` 같은 사설을 금지하고 `"출력은 JSON 블록 외에는 절대 작성하지 마라"`고 명시합니다.
4. **Few-shot(예시) 제공**: 원하는 형태의 샘플을 프롬프트 안에 최소 1개 이상 끼워 넣습니다.

#### 시각 자료 2. Few-shot(예제) 매핑 프롬프트 템플릿
```
### 지시사항: 아래 규칙에 맞춰 입력값을 변환하라.
[입력 예제]: 5월 3일 홍길동 대리 출장 승인
[출력 예제]: { "date": "2026-05-03", "name": "홍길동", "task": "출장" }

---
[실제 입력]: 5월 25일 김민지 과장 휴가 신청
[실제 출력]: (★ LLM이 예제를 모방하여 정확한 포맷으로 응답 생성)
```
*그림 설명: 프롬프트 내에 작성된 예제 블록이 실제 출력으로 어떻게 복제 및 확장되는지 직관적으로 보여주는 순서도입니다.*

### [00:25 - 00:50] 실습 진행: 회의록 데이터 구조화 실습 (25분)

#### 제공할 실습 텍스트 (회의록 일부)
```text
5월 정기 회의에서 신규 교육 프로그램 홍보 일정이 논의되었다.
홍보 문안은 김민지가 이번 주 금요일까지 초안을 작성하기로 했다.
랜딩 페이지 수정은 박준호가 다음 주 화요일까지 완료한다.
예산은 기존 안에서 10% 증액하는 방향으로 검토하기로 했다.
수강생 모집 공지는 다음 주 수요일 오전에 발송한다.
추가로 교육 만족도 조사 문항을 이서연이 정리하기로 했다.
```

#### 실습 수행 단계
1. **표 변환**: 회의록을 표(`구분, 내용, 담당자, 기한`) 컬럼에 맞춰 Markdown 표로 바꿉니다.
2. **Bullet 요약**: 핵심만 추려 핵심 요약 형태로 요점 정리합니다.
3. **JSON 변환**: 지정된 스키마 구조(`{ "decisions": [], "tasks": [ { "task": "", "owner": "", "due_date": "" } ] }`)에 맞춰 순수한 JSON으로만 답변을 인출합니다.

#### 강사용 팁 (Instructor Tip)
> "JSON으로 변환할 때 자꾸 사설이 섞이거나 괄호가 깨지는 일이 발생합니다. 이럴 때는 프롬프트 맨 아래에 `# 제약조건: 반드시 JSON 외 다른 텍스트는 금지한다.`라는 한 문장을 덧붙여 Attention을 잡아주도록 유도하세요."
