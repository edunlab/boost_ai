# [교안] 6차시. 업무 자동화 Agent 설계: Mini Project

## 1. 개요 및 목표

- **대상**: 지금까지 배운 에이전트 설계 기법을 바탕으로 자신의 실무 문제를 해결할 설계도를 작성하려는 학습자
- **시간**: 총 60분 (도입 5분, 설명 15분, 실습 30분, 발표 및 정리 10분)
- **목표**:
  - 어떤 업무가 AI 자동화에 적합한지 평가 기준(반복성, 규칙성, 데이터성)을 설명할 수 있다.
  - 문제 정의부터 입력, 처리 흐름, 조건 분기, 출력 형식을 포함하는 Agent 설계서를 작성할 수 있다.
  - 설계서를 기반으로 논리적으로 에이전트의 작동 흐름과 기대 효과를 발표하고 피드백을 수용한다.

## 3. 세부 강의 가이드

### [00:00 - 00:05] 도입: 생각 열기 (5분)

#### 강사 설명 가이드

> "마지막 6차시입니다! 이제 여러분의 손으로 직접 실무 자동화 에이전트 설계서를 만들어 보는 Mini Project 시간입니다. 에이전트를 만들 때 가장 중요한 것은 '멋있고 대단한 일'을 시키는 게 아니라, '매일 반복되면서 규칙이 뚜렷한 일'을 시키는 것입니다. 어떤 일이 에이전트의 먹잇감으로 적합한지 살펴보고 실제 설계도를 완성해 보겠습니다."

#### 도입 질문

- _"매일 아침 출근하자마자 하는 귀찮고 반복적인 텍스트 업무는 어떤 것이 있나요?"_
- _"그 일은 AI가 처리할 수 있는 명확한 기준이나 규칙이 있나요?"_

### [00:05 - 00:20] 핵심 설명: 자동화 기준 및 설계서 작성법 (15분)

#### 1) 자동화 대상 선정을 위한 3대 평가 기준

어떤 일이 에이전트 도입에 적합한지 평가하는 척도입니다. (1~5점 척도로 평가)

1. **반복성(Frequency)**: 일주일 혹은 하루에 몇 번이나 일어나는가? (자주 일어날수록 적합)
2. **규칙성(Rule-based)**: 일하는 방법과 판단의 로직을 글로 써서 다른 사람에게 설명할 수 있는가? (설명할 수 없다면 LLM도 판단하기 어려움)
3. **데이터성(Data-driven)**: 입력되는 값과 최종 뽑아내는 결과가 텍스트, 표, 엑셀 등으로 전산화되어 있는가? (구두 지시나 육체 노동은 불가)

#### 시각 자료 1. 자동화 적합성 매트릭스

```
   고 ▲
     │                 [최적의 대상 (★)]
  규 │ - 주간 보고서 요약 및 작성
  칙 │ - 반복 고객 메일 자동 답변 분류
  성 │
     ├────────────────────────────────────
  저 │ - 신규 비즈니스 전략 브레인스토밍
     │ - 디자인 아이디어 스케치
     └────────────────────────────────────
     저                 데이터성                ▲ 고
```

_그림 설명: 가로축(데이터성)과 세로축(규칙성)이 높고 반복적인 영역에 우리가 설계할 최적의 에이전트가 위치함을 보여주는 2차원 매트릭스 도표입니다._

#### 2) Agent 설계서 11개 항목 작성 양식 안내

실습 시간 동안 작성하게 될 설계 양식의 주요 골자입니다.

- **프로젝트명**: 에이전트의 명칭
- **해결하려는 문제**: 현재의 비효율성과 자동화의 필요성
- **사용자 및 대상 업무**: 이 에이전트를 조작할 사람
- **입력 데이터**: 들어갈 텍스트 또는 문서 예시
- **처리 단계 (Workflow)**: Step 1, Step 2, Step 3의 역할과 연동 방식
- **조건 분기**: 어떤 상황에 따라 처리 경로가 나뉘는지
- **출력 결과 & 활용 방식**: 결과 포맷(표, JSON 등) 및 후속 연계처
- **기대 효과**: 아낄 수 있는 시간과 비용적 가치

#### 시각 자료 2. 에이전트 전체 설계도 매핑 (Architecture Map)

```
[ Input: 로우 데이터 ] ──> [ Step 1: 분류 ] ──(분기)──> [ Step 2-A: 긍정 템플릿 ] ──> [ Output: 표/JSON ]
                                      └──> [ Step 2-B: 부정 알림 ]
```

_그림 설명: 입력부터 여러 단계의 워크플로우와 조건 분기를 거쳐 최종 포맷팅된 결과로 출력되는 구조가 유기적으로 매핑된 시스템 설계도입니다._

### [00:20 - 00:50] 실습 진행: Agent 설계서 및 발표 자료 작성 (30분)

#### 실습 단계

1. **업무 후보 도출**: 팀원들 또는 개인별로 자동화하고 싶은 후보 업무를 3개 적습니다.
2. **적합성 평가 및 선정**: 3대 기준(반복성, 규칙성, 데이터성)으로 점수를 매겨 가장 점수가 높은 1개 업무를 선택합니다.
3. **설계서 작성**: 선정된 업무에 대해 제공된 에이전트 설계서 양식의 항목을 꼼꼼하게 채웁니다.
4. **발표용 1페이지 요약본 작성**: 동료들에게 설명하기 위한 3분 스피치 요약 카드를 완성합니다.

#### 강사용 팁 (Instructor Tip)

> "학습자들이 기술적 구현(코딩)에 대해 걱정하지 않도록 해 주세요. '말과 표로 일을 논리적으로 정리해 두는 것'이 에이전트 기획의 90% 이상임을 지속적으로 안심시키고 독려하는 것이 핵심입니다."

### [00:50 - 01:00] 발표 및 총정리 (10분)

#### 팀별 발표 가이드

- 팀별로 **3분** 동안 핵심만 명확히 발표하게 합니다.
  - 1분: 어떤 문제고 왜 자동화가 필요한가? (문제 정의)
  - 1분: 입력 데이터와 처리 흐름(Workflow, 조건 분기)은 어떻게 되는가?
  - 1분: 최종 결과물은 무슨 포맷이고 어떤 효과를 기대하는가?

#### 평가 루브릭 (Peer Feedback Rubric)

- **우수**: 문제가 구체적이고, 입력/처리/조건/출력이 유기적으로 연동되며, 효과가 정량적으로 묘사됨.
- **보완 필요**: 문제가 너무 모호하거나, 조건 분기가 생략되었거나, 결과 포맷이 정해지지 않아 실무 적용이 어려움.
