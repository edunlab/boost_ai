# [교안] 1차시. AI · LLM · Agent 개념 이해

## 1. 개요 및 목표

- **대상**: AI를 처음 접하거나 개념적 정리가 필요한 비전공자/초심자
- **시간**: 총 60분 (도입 5분, 설명 20분, 실습 25분, 공유 및 정리 10분)
- **목표**:
  - AI, LLM, Agent의 관계를 일상적인 비유를 통해 쉽게 설명할 수 있다.
  - ChatGPT를 사용하는 일반적인 대화와 업무를 처리하는 Agent 설계의 차이를 이해한다.
  - 에이전트의 4대 구성요소인 입력(Input), 판단(Reasoning), 행동(Action), 출력(Output)을 설명할 수 있다.

### [00:05 - 00:25] 핵심 설명: 초심자를 위한 개념 정리 (20분)

#### 1) AI · LLM · Agent의 관계 (비유로 이해하기)
초심자가 이해하기 어렵지 않도록 전문 용어를 배제하고 다음과 같은 비유로 설명합니다.

- **AI (인공지능) ──> [비유: 과학 기술이라는 거대한 숲]**
  - 기계가 인간처럼 생각하고 판단하게 만드는 모든 기술을 뜻하는 가장 넓은 범위입니다.
- **LLM (대형 언어 모델) ──> [비유: 도서관의 책을 모두 읽은 똑똑한 '뇌']**
  - 엄청나게 많은 글을 읽고 학습하여, 문장을 자연스럽게 이어 쓰는 능력을 가진 핵심 엔진(뇌)입니다. (예: GPT-4, Claude 등)
- **Agent (에이전트) ──> [비유: 뇌(LLM)에 팔, 다리(Action)와 행동 규칙을 달아준 '일꾼 로봇']**
  - 단순히 말만 하는 것이 아니라, 주어진 목표를 달성하기 위해 '스스로 계획을 세우고, 행동(도구 사용)하고, 결과를 만들어내는 완성된 시스템'입니다.

#### 시각 자료 1. AI · LLM · Agent 관계 다이어그램
```
+-------------------------------------------------------+
| AI (인공지능: 컴퓨터 과학의 거대한 숲)                |
|  +-------------------------------------------------+  |
|  | LLM (대형 언어 모델: 똑똑하게 생각하는 '뇌' 엔진)|  |
|  |  +-------------------------------------------+  |  |
|  |  | Agent (에이전트: 행동력과 규칙을 갖춘 로봇)|  |  |
|  |  +-------------------------------------------+  |  |
|  +-------------------------------------------------+  |
+-------------------------------------------------------+
```
*그림 설명: 세 개의 네모 또는 원이 서로를 포함하는 구조(AI > LLM > Agent)를 통해 에이전트가 LLM을 기반으로 구축되는 실무형 시스템임을 보여줍니다.*

#### 2) 에이전트의 4대 구성요소: IRAO (아주 중요!)
에이전트가 일하는 방식은 사람이 일하는 방식과 똑같습니다. 다음 4단계를 거칩니다.

1. **Input (입력)**: 일거리 주기 (예: 고객이 보낸 이메일)
2. **Reasoning (판단/추론)**: 뇌로 생각하기 (예: "화난 고객이네? 배송 지연 때문이군. 사과하고 해결책을 찾아야겠다.")
3. **Action (행동/도구)**: 손발로 일하기 (예: 택배 시스템에서 배송 상태 조회하기, 이메일 답장 발송하기)
4. **Output (출력)**: 결과물 내놓기 (예: 작성된 사과 답장 메일과 데이터베이스 처리 기록)

#### 시각 자료 2. 에이전트의 4대 구성요소 (IRAO) 흐름도
```
[사용자 요청 / 데이터] ──> ( 뇌: 판단/추론 ) ──> [ 도구 실행 / API ] ──> [ 최종 업무 산출물 ]
      (Input)                 (Reasoning)               (Action)                 (Output)
```
*그림 설명: 입력부터 출력까지 왼쪽에서 오른쪽으로 흐르는 파이프라인 형태의 인포그래픽으로, 에이전트가 중간에 판단과 도구 실행 과정을 반드시 거친다는 것을 시각적으로 표현합니다.*

### [00:25 - 00:50] 실습 진행: 질문 방식별 응답 비교 실습 (25분)

#### 실습 목표
- 프롬프트에 구체적인 '행동 방식(Reasoning)'과 '출력 양식(Output)'을 정해주었을 때 답변이 얼마나 실무적으로 변하는지 직접 경험해봅니다.

#### 제공할 실습 텍스트 (고객 불만 이메일)
```text
제목: 배송 지연에 대한 항의

안녕하세요, 지난주 주문한 제품이 아직도 배송되지 않았습니다.
주문 시 2-3일 내 배송이라고 했는데 벌써 일주일이 지났습니다.
연락도 없고, 배송 조회도 안 됩니다. 환불해주세요.
```

#### 실습 단계 (학습자 개별 수행)

- **1단계 (일반 질문)**: 아무런 옵션 없이 기본 질문만 던집니다.
  - 프롬프트: `"고객 불만 메일에 답장해줘."`
- **2단계 (역할 지정)**: AI에게 구체적인 직업/역할(Persona)을 줍니다.
  - 프롬프트: `"너는 친절한 고객지원 담당자야. 고객 불만 메일에 답장해줘."`
- **3단계 (구조화 질문)**: 일하는 순서(Reasoning)와 어떻게 응답할지 형식(Output)을 강제합니다.
  - 프롬프트: `"너는 고객지원 전문가야. 고객 불만 메일을 분석하고, 불만 원인, 사과 문장, 해결 방안, 후속 조치로 나누어 답장 초안을 작성해줘."`

#### 강사용 팁 (Instructor Tip)
> "학습자들이 직접 Antigravity 프롬프트 테스트 창에 세 가지를 각각 입력해 보도록 하세요. 줄글로 대충 써주던 1단계에 비해, 3단계에서는 메일이 깔끔하게 부서별 조치 계획이나 원인 분석 항목으로 구조화되어 출력되는 것을 관찰하도록 유도합니다."
