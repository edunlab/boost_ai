# [교안] 4차시. Agent 흐름 설계: Prompt에서 Workflow로

## 1. 개요 및 목표

- **대상**: 복잡한 비즈니스 문제를 체계적이고 안정적인 파이프라인으로 구조화하려는 학습자
- **시간**: 총 60분 (도입 5분, 설명 20분, 실습 25분, 공유 및 정리 10분)
- **목표**:
  - 복잡한 지시를 하나의 긴 프롬프트에 담을 때 발생하는 한계점(Attention 분산)을 이해한다.
  - 프롬프트 체이닝(Prompt Chaining)의 개념과 구성 원리를 설명할 수 있다.
  - TSF(Task, Step, Flow) 기법을 활용하여 3단계(요약-핵심 이슈 추출-액션 아이템 생성) 워크플로우를 설계한다.

## 3. 세부 강의 가이드

> "인턴 사원에게 출근 첫날 '우리 팀 지난주 실적 요약해 주고, 문제점 두 개 뽑아서, 다음 회의에서 쓸 발표 장표 기안서까지 작성해서 이메일로 보내줘'라고 한 번에 지시하면 어떻게 될까요? 높은 확률로 어딘가 누락하거나 엉망으로 해올 것입니다. AI도 똑같습니다. 한 번에 너무 많은 지시를 내리면 집중력이 흩어져 일을 그르칩니다. 이를 해결하기 위한 정답은 '일을 쪼개고 순서대로 시키는 것(Workflow)'입니다."

### [00:05 - 00:25] 핵심 설명: 프롬프트 체이닝과 TSF 프레임워크 (20분)

#### 1) 프롬프트 체이닝(Prompt Chaining) 개념
- **정의**: 하나의 작업을 단계별로 쪼개어, **앞 단계의 결과물(Output)을 다음 단계의 입력값(Input)**으로 바톤 터치하며 연결해 나가는 다단계 실행 구조입니다.
- **핵심 장점**:
  - 각 단계마다 전용 페르소나와 명확한 지시어를 매핑하여 출력 품질이 비약적으로 향상됩니다.
  - 중간 과정이 마음에 안 들면 그 단계만 수정(디버깅)할 수 있어 관리가 편합니다.
  - 중간에 사람의 승인/수정 단계를 끼워 넣는 것(Human-in-the-loop)이 가능합니다.

#### 시각 자료 1. 바톤 터치형 프롬프트 체이닝 구조도
```
[원문 텍스트] ──> (Step 1: 요약) ──> [요약 결과]
                                           │
                                           ▼
                                    (Step 2: 이슈 분석) ──> [핵심 이슈]
                                                                │
                                                                ▼
                                                         (Step 3: 대책 수립) ──> [최종 계획]
```
*그림 설명: 육상 경기에서 바톤을 주고받는 릴레이 주자들의 이미지에 데이터 흐름을 대입하여, 단계별 연동 파이프라인의 핵심을 시각화합니다.*

#### 2) Workflow 설계의 3요소: TSF
- **Task (과업)**: 해결하려는 전체적이고 궁극적인 업무 목표 (예: 주간 마케팅 분석 보고서)
- **Step (단계)**: 과업을 완료하기 위해 잘게 쪼갠 개별 지휘관(프롬프트) 단계
- **Flow (흐름)**: 각 Step이 수행되는 논리적 순서 및 분기 (예: A 단계 완료 후 B 실행)

#### 시각 자료 2. TSF 설계 템플릿
```
+------------------------------------------------------------+
| TASK: 주간 고객 피드백 분석                                |
+------------------------------------------------------------+
|  [Step 1] 요약가 ────> [Step 2] 분석가 ────> [Step 3] 전략가  |
|  - 요약문 출력         - 감정 분류           - 개선안 표 출력 |
+------------------------------------------------------------+
```
*그림 설명: 하나의 전체 공장 라인(Task) 안에 세부 공정 기계(Step)들이 컨베이어 벨트(Flow)로 연결되어 있는 직관적인 개념도입니다.*

### [00:25 - 00:50] 실습 진행: 3단계 워크플로우 설계 및 체이닝 (25분)

#### 실습 업무
앞선 회의록 원문을 이용하여 순차적으로 3단계 에이전트 라인을 구축해 봅니다.

- **1단계: 요약 (Summarizer Persona)**
  - 입력: 회의록 원문
  - 지시: 주요 의안을 200자 이내로 핵심만 압축
  - 출력: Bullet Point 3줄
- **2단계: 핵심 이슈 분석 (Analyst Persona)**
  - 입력: **1단계 요약 결과**
  - 지시: 요약 결과에서 업무 병목을 일으킬 만한 지연 요소를 분석
  - 출력: JSON 데이터 `{ "issues": [] }`
- **3단계: 액션 아이템 도출 (Strategist Persona)**
  - 입력: **2단계 JSON 데이터**
  - 지시: 분석된 각 이슈별로 구체적으로 해야 할 담당자와 타임라인 제안
  - 출력: Markdown 표

#### 강사용 팁 (Instructor Tip)
> "학습자들이 복사-붙여넣기를 통해 1단계 결과물을 2단계 프롬프트의 컨텍스트로 수동 조립해 보도록 하세요. 실제 프로그램 코드가 뒤에서 어떻게 이 데이터를 실어 나르는지 직접 체감하게 하는 것이 목적입니다."
