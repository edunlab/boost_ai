도입 질문: 줄글 응답의 한계
1일차 에듀니티랩
  • 도입 질문: "아무리 훌륭한 보고서 요약본이라도, 복사해서 공유하거나 ERP 시스템에 바로 등록하기 어려운 근본적인 이유는 무엇일까요?"
  • 문제점 진단:
    • 가독성 저하: 긴 텍스트 덩어리는 핵심 파악에 시간이 걸림.
    • 재사용 불가: 텍스트 사이에 숨어 있는 일정, 담당자 정보를 컴퓨터가 자동으로 추출해 내기 어려움.
    • 비정형성의 한계: 실행할 때마다 포맷이 달라져 후속 프로세스 자동화가 불가능함.
인간에게 유용한 지식은 정제되고 규격화된 상태로 존재합니다. 만약 AI가 쓴 보고서 요약본이 매번 서술 방식이 달라진다면, 그것은 자동화가 아니라 또 다른 수작업의 시작입니다. 비즈니스 에이전트는 출력의 포맷을 강하게 고정해야만 합니다.
정리되지 않은 뒤엉킨 텍스트 더미가 필터를 통과하면서 깔끔한 스프레드시트로 정돈되는 모션 다이어그램.

핵심 설명: 3대 출력 포맷 (Table, Bullet, JSON)
1일차 에듀니티랩
Table (표 형식) Markdown Table
비교 분석, 담당자 배정, 일정 관리 등 인간의 보고서 및 엑셀 정리에 최적화된 형식입니다.
Bullet Point (불릿 형식) List
빠른 핵심 요약 전달, 주간 보고 브리핑, 메신저 공유에 최적화된 형식입니다.
JSON (데이터 형식) System Data
백엔드 데이터베이스 저장, 외부 API 전송 및 타 에이전트 시스템 연동에 필수적인 구조적 코드 형식입니다.
우리는 목적에 맞게 3대 포맷을 골라 써야 합니다. 보고를 위해서는 표나 불릿이 좋지만, 데이터베이스에 넣거나 다음 에이전트로 값을 넘길 때는 컴퓨터가 파싱하기 쉬운 JSON 형식을 적극 사용해야 자동화가 매끄러워집니다.
세 개의 열을 나누어 각각 표 화면, 불릿 텍스트, JSON 코드 블록 예시를 깔끔하게 배치하여 가독성을 높임.
3차시. 출력 구조 제어 & 데이터화
핵심 설명: 출력 구조 제어 프롬프트 기법
1일차 에듀니티랩
  • 출력 형식을 강제하는 4대 프롬프트 규칙:
    • 1. 스키마 정의: 출력해야 할 항목의 이름을 명확히 기술 (예: 구분, 내용, 담당자, 기한)
    • 2. 빈 값 처리 규칙: 정보가 누락되었을 때의 대응 방안 규정 (예: "기한이 없으면 'N/A'로 표기")
    • 3. 불필요 어구 금지: 챗봇의 인사말이나 사설 차단 (예: "오직 JSON만 출력하고 부가 텍스트는 금지한다")
    • 4. 예시 제공 (Few-shot): 원하는 최종 아웃풋의 템플릿 예제를 제공합니다.
LLM이 자기 마음대로 친절한 안내 문구를 덧붙이는 순간, 후속 프로그램은 에러를 뿜어냅니다. 이를 막기 위해 스키마 정의와 함께 '오직 JSON만 출력하고 부가 설명은 생략하라'는 강력한 제어 지시를 삽입해야 합니다.
'가드레일 지시어'와 'Few-shot 템플릿' 예제가 하이라이트 표시된 프롬프트 코드 구조도.

3차시. 출력 구조 제어 & 데이터화
실무 팁: 안정적인 JSON 파싱 기법
1일차 에듀니티랩
  • JSON 파싱 안정성 확보 팁:
    • 1. JSON Markdown Block 유도: AI에게 ```json ... ``` 안에만 JSON을 출력하도록 하여 프로그램이 파싱하기 쉽게 만듭니다.
    • 2. Key명 영문 고정: 키 값을 영문(e.g., owner, due_date)으로 고정해야 나중에 에러가 없습니다.
    • 3. 따옴표 이스케이프 명시: 텍스트 내의 쌍따옴표로 인한 JSON 구조 파손을 막기 위해 "문장 속 따옴표는 이스케이프할 것"이라는 룰을 심습니다.
실무에서 JSON 데이터를 파싱하다 보면 따옴표 하나 때문에 전체 에러가 나서 시스템이 멈추는 일이 빈번히 생깁니다. 이를 예방하기 위해 프롬프트 내에 키 값의 영문 명칭을 규정하고, 따옴표나 줄바꿈 문자의 처리 규칙을 추가해 주는 것이 정석입니다.
JSON 에러 예시 코드와 이를 예방하는 수정 코드를 나란히 띄워 차이점을 시각적으로 설명.
