2일차 에듀니티랩
  • Reasoning 출력의 가치:
    • AI가 분류 결과를 낼 때 그 근거(이유)를 함께 출력하도록 프롬프트를 설계함.

AI에게 '그냥 결과만 말해'라고 하면 검증할 방법이 없습니다. 반드시 '왜 그렇게 생각했는지 이유를 적어줘'라고 지시해야 합니다. 이 판단 이유는 나중에 강사나 운영진이 에이전트의 로직을 튜닝할 때 엄청난 힌트가 되며, 현업 실무자에게도 신뢰감을 줍니다.
JSON 응답 코드 내부에서 `"reason"` 키를 돋보기 기호로 크게 확대해 보여주는 연출.

5차시. 조건 분기 & 판단형 Agent 설계
실습 진행: 리뷰 데이터셋 및 가이드라인
2일차 에듀니티랩
Python Self-Correction 예외 처리 예제 
import json
import openai

def get_valid_json(prompt, max_retries=3):
    current_prompt = prompt
    for attempt in range(max_retries):
        response = openai.chat.completions.create(model="gpt-4o", messages=[{"role": "user", "content": current_prompt}])
        raw_text = response.choices[0].message.content
        try:
            # JSON 검증 시도
            data = json.loads(raw_text)
            return data
        except json.JSONDecodeError as e:
            print(f"Error parsing JSON: {e}. Retrying...")
            # 에러 피드백을 전달하여 다시 생성을 요청함
            current_prompt = f"{prompt}\n\n[이전 시도 에러 발생]: {e}\n위의 에러를 반영하여 완벽한 JSON을 다시 작성하시오."
    raise Exception("Max retries reached without valid JSON output.")
5개의 리뷰 텍스트 상자 옆에 각각 카테고리 꼬리표(Tag)가 정답 가이드라인과 매칭되는 시각적 구성.

많은 조에서 4번 리뷰의 처리를 두고 논의하셨을 겁니다. '설명은 좋았다'는 긍정이지만 '실습이 부족했다'는 개선 요구(부정)입니다. 이를 '복합'으로 똑똑하게 잡아내고 불편 요소를 우선하여 후속 조치를 취하도록 설계했다면 훌륭한 수준의 에이전트 설계입니다.
의견이 갈리는 토론 지점을 저울(Scale) 그래픽으로 묘사하여 판단 기준의 미세 튜닝 중요성 부각.
